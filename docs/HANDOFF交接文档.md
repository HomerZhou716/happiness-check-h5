5. **「幸福驾校」三科目 H5**（当前）——把 xlsx 的量表接回纸境 H5：科目一 PERMA（原个人版降级为此）+ 科目二夫妻 + 科目三亲子 + 幸福驾照。**这是当前主线。**

## 3. 关键决策与理由（决策日志）

| 决策 | 理由 |
|---|---|
| 主视觉选「纸境」 | 项目负责人审美偏好——东方文人纸墨风，非疗愈渐变/深色编辑风 |
| 从泛幸福转向家庭定位 | 多方商业评审一致：泛"测幸福"需求认可度仅约 27%、排第 5；关系/亲子是付费级痛点 |
| 55 题砍到 24 题 | 移动端 30–45 分钟问卷完成率断崖；15–25 题是安全区。砍掉性别预设题/社会赞许题/条件题/重复题 |
| A/B/C 3 点而非 7 点 | 家庭题是行为频率题，天然三锚点；答题快，选项本身是传播素材。科目一 PERMA 保留 7 点不动 |
| 选项显示顺序随机 | 3 点计分若"永远第一个满分"会被学会刷分；随机化是保住 3 点可用性的前提 |
| 等级用"驾照"话术 | 原五档考试式命名（优秀/需重视）浪费隐喻且踩合规；改教练级/老司机/实习期/陪练期，对行为不对人 |
| 商业方向 = B2B2C | 纯 C 端无流量/无背书/买量算不过账；唯一活路是企业员工家庭关怀福利。详见内部商业方案 |

## 5. 坑 / 注意事项（务必读）

1. **屏幕切换门闩不能用 rAF**——页面不可见时 rAF 不触发会卡死交互。用 `setTimeout` 同步释放。修过一次，别改回去。
2. **只改 `variants/paper/`**——`demo/` 和其他 variants 是留档，改了污染历史对照。
3. **仓库是 public**——提交前扫无密钥/无敏感。GitHub Pages 免费版要求 public，所以别为省事把整仓设私有（Pages 会挂）。
4. **github.io 微信打不开**——分享给团队/客户用普通浏览器没问题；正式对 C 端用户投放**必须**迁国内托管（腾讯云 COS/CDN 等）。
5. **git 提交身份**——目前是本机默认 `Homer <logion@Homer-Mac-Studio.local>`，且有未配置警告。接手后建议 `git config user.name/email` 配成你自己的。
6. **改题库会破坏哈希测试**——`subjects.js` 的题干/选项是逐字定稿，改动需同步更新 `app.test.js` 里的规范哈希，且要有心理测量依据。
7. **禁词红线**——任何用户可见文案不得出现诊断/治疗/治愈/抑郁/焦虑症/心理医生/疗愈/寻求专业帮助。
