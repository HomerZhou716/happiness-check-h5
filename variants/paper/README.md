# 幸福驾校 · 纸境版

纯静态、零外部依赖的移动端 H5。视觉延续纸境：宣纸暖白、墨色正文、朱砂印章与描金分隔线。

## 产品结构

- 科目一 · 自我状态：原 PERMA 15 题、7 点量表、五维报告、7 天微训练与 60 秒呼吸练习。
- 科目二 · 夫妻同行：12 道三选一行为题，覆盖情感连接、同盟与决策、冲突与修复。
- 科目三 · 亲子上路：12 道三选一行为题，覆盖亲密陪伴、尊重与回应、双教练协同。
- 幸福驾照：科目二和科目三都完成后出现，以两科均分生成领证分、等级、扣分总数、颁发日期和复检路线。

科目二、三各自拥有独立报告。报告展示三个子维度得分条、最弱子维度解读、全部 0 分项，以及四个指定行为命中时才点亮的危险驾驶红灯。报告会按最弱子维度进入对应 7 天陪练，Day 1 统一复用原 60 秒呼吸练习。

## 文件职责

- `index.html`：三科目入口及全部屏幕语义结构。
- `style.css`：纸境视觉、移动端布局、动效与减少动效适配。
- `subjects.js`：科目二/三的 24 题、6 个子维度、6 套陪练、等级与复检数据。
- `app.js`：PERMA 原流程、家庭科目流程、计分、状态恢复、报告、陪练与驾照渲染。
- `app.test.js`：使用 Node 内置测试能力运行的数据与逻辑契约。
- `IMPLEMENTATION_PLAN.md`：本次三科目升级的实施与验收记录。

`index.html` 必须先加载 `subjects.js`，再加载 `app.js`。

## 计分规则

科目二/三每题三个语义选项固定为 2、1、0 分。显示顺序逐题随机，随机顺序随进度一起保存，因此视觉位置不会改变语义分值。

- 子维度分：4 题合计除以满分 8，再四舍五入到百分制。
- 科目分：3 个子维度百分制分数的均值，四舍五入。
- 领证分：科目二、科目三各占 50%，均值四舍五入。
- 驾照等级：85 分起为教练级，70 分起为老司机，55 分起为实习期，其余为陪练期。
- 驾照扣分总数：`100 - 领证分`。
- 报告中的每个 0 分项显示为“扣 8 分”，用于形成可执行的补练清单；危险红灯不额外改变数值分数。

## 本地存储

所有结果默认只保存在当前设备的浏览器本地：

- 科目一：`happiness-self-check:paper:v1`
- 科目二：`happiness-driving:s2:v1`
- 科目三：`happiness-driving:s3:v1`

重测某一科只清除该科数据，不影响另外两科。重测科目二或三后，在两科重新完成前，领证入口会立即收回。

## 运行与自检

页面没有构建步骤，可直接打开 `index.html`。开发验收命令：

```sh
node --test app.test.js
node --check subjects.js
node --check app.js
```

推荐按 390×844 检查完整流程，并补查 320×568、430×932 与 700px 以上桌面宽度。重点验证：刷新恢复、返回上一题、随机顺序固定、快速连点门闩、四个红灯、六套 Day 1 呼吸入口、两科完成后领证，以及科目一完整回归。

本测评仅用于生活状态的自我观察，不替代专业支持或个性化建议。
